# Product Requirements Document (PRD)
## El Coreano

### 1. Overview
**Product Name:** El Coreano  
**Domain:** elcoreano.com  
**Platform:** Web  
**Description:**  
A Reddit-style community platform where Spanish-speaking users can learn Hangul, share study materials, and explore Korean culture.

### 2. Objectives
- Provide a collaborative environment for learning Hangul.
- Prevent learners from giving up by fostering community engagement.
- Enable users to share study resources and tips.

### 3. Target Users
- Spanish-speaking beginners from Latin America.
- K-POP and K-Drama fans.
- Age group: 20–40.

### 4. Core Features (MVP)
- Email-based authentication.
- Community discussion boards.
- Create, edit, and delete posts.
- Comment system.
- Category-based navigation.
- Search functionality.
- Admin moderation tools.
- Spanish language interface.

### 5. Categories
- Alfabeto Coreano
- Pronunciación
- Vocabulario
- Oraciones
- Recursos de Aprendizaje
- Preguntas y Respuestas
- Aprende con K-POP
- Aprende con K-Dramas
- Memes y Tendencias de Corea
- Comunidad General

### 6. Technology Stack
- Frontend: Antigravity (Next.js)
- Backend: Supabase
- Database: PostgreSQL
- Authentication: Supabase Auth (Email)
- Storage: Supabase Storage
- Deployment: Vercel

### 7. Design Requirements
- Clean and minimalist interface.
- Inspired by Reddit.
- Mobile-responsive design.
- Spanish as the primary language.

### 8. Success Metrics
- 1,000 users within 3 months of launch.
- 100 Daily Active Users (DAU).
- 300 posts created within 3 months.
- 30% user retention rate.

### 9. Timeline
- MVP Launch: Within 100 days.

### 10. Monetization
- Free service during MVP phase.
- Potential future monetization through premium features or ads.
